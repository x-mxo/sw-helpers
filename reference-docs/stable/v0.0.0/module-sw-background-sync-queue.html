sw-background-sync-queue

Queues failed requests and uses the Background Sync API to replay those requests at a later time when the network state has changed.

Classes

BackgroundSyncQueue

Instance Methods

(static) this.getResponse(id) → Object

This function returns the fetched response for the given id of the request

Parameters
id String

The ID of the request given back by the broaadcast channel
Returns Object

response Fetched response of the request.

(static) this.initialize(input)

In order to use this library call goog.backgroundSyncQueue.initialize(). It will take care of setting up IndexedDB for storing requests and broadcast channel for communication of request creations. Also this attaches a handler to sync event and replays the queued requeusts.

Parameters
input Object (Optional)

The input object to this function

Properties
dbName string (Optional)

The name of the db to store requests and responses
