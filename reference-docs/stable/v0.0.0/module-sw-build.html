sw-build

To get a list of url's and a corresponding revision details, call either generateFileManifest or getFileManifestEntries().

generateFileManifest() will run globs over a directory of static assets and write a JavaScript file containing URL's and revision details for those files.

If you'd rather receive the data as a JavaScript Array, use getFileManifestEntries() instead.

If you only need precaching of your static assets in your service worker and nothing else, you can generate complete service worker with generateSW().

Examples

Generate a build manifest file.

const swBuild = require('sw-build');

swBuild.generateFileManifest({
  globDirectory: './build/',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['service-worker.js','admin.html'],
  dest: './build/scripts/manifest.js',
  templatedUrls: {
    '/shell': ['shell.hbs', 'main.css', 'shell.css'],
  },
})
.then(() => {
  console.log('Build file has been created.');
});

Get a list of files with revision details.

const swBuild = require('sw-build');

swBuild.getFileManifestEntries({
  globDirectory: './build/',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['service-worker.js','admin.html'],
  templatedUrls: {
    '/shell': ['shell.hbs', 'main.css', 'shell.css'],
  },
})
.then((fileDetails) => {
  // An array of file details include a `url` and `revision` parameter.
});

Generate a service worker for a project.

const swBuild = require('sw-build');

swBuild.generateSW({
  globDirectory: './build/',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['service-worker.js','admin.html'],
  dest: './build/sw.js',
  templatedUrls: {
    '/shell': ['shell.hbs', 'main.css', 'shell.css'],
  },
})
.then(() => {
  console.log('Service worker generated.');
});

Instance Methods

(static) this.generateFileManifest(input) → Promise

Example

Generate a build manifest of static assets, which could then be used with a service worker.

const swBuild = require('sw-build');

swBuild.generateFileManifest({
  dest: './build/manifest.js'
  globDirectory: './build/',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['admin.html'],
  format: 'iife', // alternatively, use 'es'
})
.then(() => {
  console.log('Build Manifest generated.');
});

This method will generate a file manifest that can be used in a service
worker for caching assets offline.
Parameters
input Object
Properties
dest String

The name and path you wish to write your manifest file to.
globDirectory String

The root of the files you wish to be cached. This will also be the directory the service worker and library files are written to.
staticFileGlobs Array.<String>

Patterns to glob for when generating the build manifest.
globIgnores String | Array.<String> (Optional)

Patterns to exclude when generating the build manifest.
templatedUrls Object.<String, (Array|String)> (Optional)

If a URL is rendered/templated on the server, its contents may not depend on a single file. This maps URLs to a list of file names, or to a string value, that uniquely determines each URL's contents.
format String (Optional)

Default format is 'iife', but also accepts 'es', which outputs an ES2015 module.
Returns Promise

Resolves once the service worker has been generated with a precache list.

(static) this.generateSW(input) → Promise

Example

Generate a service worker for a project.

const swBuild = require('sw-build');

swBuild.generateSW({
  globDirectory: './build/',
  dest: './build/sw.js',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['admin.html'],
  templatedUrls: {
    '/shell': ['shell.hbs', 'main.css', 'shell.css'],
  },
})
.then(() => {
  console.log('Service worker generated.');
});

This method will generate a working service worker with an inlined
file manifest.
Parameters
input Object
Properties
globDirectory String

The root of the files you wish to be cached. This will also be the directory the service worker and library files are written to.
staticFileGlobs Array.<String>

Patterns to glob for when generating the build manifest.
globIgnores String | Array.<String> (Optional)

Patterns to exclude when generating the build manifest.
dest String

The name you wish to give to your service worker file.
templatedUrls Object.<String, (Array|String)> (Optional)

If a URL is rendered/templated on the server, its contents may not depend on a single file. This maps URLs to a list of file names, or to a string value, that uniquely determines each URL's contents.
modifyUrlPrefix String (Optional)

An optional object of key value pairs where the key will be replaced at the start of a url with the corresponding value.

An optional ID to be prepended to caches used by sw-build. This is primarily useful for local development where multiple sites may be served from http://localhost.
handleFetch Boolean (Optional)

Stops the generated service worker from handling fetch events, i.e. everything goes to the network. (Defaults to true.)
skipWaiting Boolean (Optional)

An optional boolean that indicates if the new service worker should activate immediately (Defaults to false).
clientsClaim Boolean (Optional)

An optional boolean that indicates if the new service worker should claim current pages (Defaults to false).
directoryIndex string (Optional)

An optional string that will append this string to urls ending with '/' (Defaults to 'index.html').

Resolves once the service worker has been generated with a precache list.

(static) this.injectManifest(input) → Promise

Example

Generate a build manifest of static assets, which could then be used with a service worker.

const swBuild = require('sw-build');

swBuild.injectManifest({
  globDirectory: './build/',
  staticFileGlobs: ['**\/*.{html,js,css}'],
  globIgnores: ['admin.html'],
  swSrc: './src/sw.js',
  swDest: './build/sw.js',
})
.then(() => {
  console.log('Build Manifest generated.');
});

This method will read in an existing service worker file and replace an empty
array in a call like so: `.precache([])`, to an array of files
with up to date array revision details. This allows the service worker
to efficiently cache assets that will be available offline.
Parameters
input Object
Properties
swDest String

The name and path you wish to write your manifest file to.
globDirectory String

The root of the files you wish to be cached. This will also be the directory the service worker and library files are written to.
staticFileGlobs Array.<String>

Patterns to glob for when generating the build manifest.
globIgnores String | Array.<String> (Optional)

Patterns to exclude when generating the build manifest.
templatedUrls Object.<String, (Array|String)> (Optional)

If a URL is rendered/templated on the server, its contents may not depend on a single file. This maps URLs to a list of file names, or to a string value, that uniquely determines each URL's contents.
swSrc String

File name for service worker file to read in and alter.
Returns Promise

Resolves once the service worker has been generated with a precache list.
